# 수동 업데이트 절차

이 문서는 문서 편집기에서 저장한 내용을 실제 실행앱에 반영하는 방법을 정리한 것입니다.

## 언제 사용하나요?

- `http://localhost:8000/` 편집기에서 문서를 수정한 뒤 실제 실행앱에 반영할 때 사용합니다.
- 편집기 저장은 로컬 `data.js`와 `custom_edits.json`까지만 저장합니다.
- 실제 실행앱 반영은 `수동업데이트_실행기.bat`를 실행해야 완료됩니다.

## 전체 흐름

```text
앱_실행하기.bat 실행
→ http://localhost:8000/ 편집기에서 문서 수정
→ 편집기에서 저장
→ 수동업데이트_실행기.bat 실행
→ GitHub main으로 push
→ Cloudflare build 확인
→ 실행앱에서 반영 확인
```

## 처음부터 차근차근

1. `앱_실행하기.bat`를 실행합니다.
2. 브라우저에서 `http://localhost:8000/`가 열리면 수정할 문서를 선택합니다.
3. `문서 편집`을 누르고 내용을 수정합니다.
4. `저장`을 누릅니다.
5. 저장 완료 메시지가 나오면 편집기 창은 그대로 두거나 닫아도 됩니다.
6. 앱 폴더에서 `수동업데이트_실행기.bat`를 실행합니다.
7. 창에 오류가 없고 `Manual update pushed to GitHub main.` 메시지가 나오면 GitHub 업로드가 끝난 것입니다.
8. Cloudflare Dashboard에서 `pharmacy-guideline-app`의 build가 성공했는지 확인합니다.
9. 실행앱을 새로고침해서 수정 내용이 보이는지 확인합니다.

실행앱 주소:

```text
https://pharmacy-guideline-app.olerose-parosc.workers.dev/
```

## Cloudflare에서 확인할 것

Cloudflare build 설정은 아래처럼 유지합니다.

```text
Build command
python build_dist.py
```

```text
Deploy command
npx wrangler deploy
```

```text
Root directory 또는 Path
/
```

주의: `Path` 칸에는 `Root directory /`가 아니라 `/`만 입력해야 합니다.

## 데이터/PDF 추가 자동 업데이트

PDF 또는 원본 데이터 파일을 추가하거나 바꾸는 경우에는 `자동업데이트_실행기.bat`를 켜둡니다.

- 감시 폴더: `C:\Users\duih\Desktop\코딩\규정 지침 앱`
- PDF 변경 감지 시 자동으로 `data.js`를 재생성합니다.
- `dist`를 다시 만들고 GitHub `main`으로 push합니다.

## 문제가 생겼을 때

- `수동업데이트_실행기.bat` 창에서 오류가 나오면 그 메시지를 확인합니다.
- Cloudflare에 `Latest build failed`가 보이면 build log를 확인합니다.
- 실행앱이 바로 바뀌지 않으면 몇십 초 뒤 새로고침합니다.
- build가 성공했는데도 안 바뀌면 Cloudflare Deployments에서 최신 `main` 버전이 100% traffic인지 확인합니다.
